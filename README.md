# Welcome to DevGuru

Welcome to DevGuru, a comprehensive repository dedicated to unraveling the curious cases and challenges encountered in the realm of design verification development across various programming languages and practices. Our mission is to illuminate the peculiar, the tricky, and sometimes the outright baffling aspects of development that we've encountered along the way.

## About This Project

DevGuru serves as a collective knowledge base where we document, discuss, and dissect issues that have puzzled, halted, or intrigued us during our development journeys. From intricate code snippets to discussions on specific commits, this project aims to be a resource for developers seeking insights or facing similar hurdles.

### Features

- __Curated Issues__: Each issue in DevGuru is a carefully documented case, complete with context, problem statements, and, when applicable, code snippets that illustrate the challenge.
- __Language Diversity__: Our repository covers a range of programming languages, reflecting the multifaceted nature of design verification development.
- __Commit References__: Some issues are tied directly to specific commits, providing a snapshot of the problem and the solution in action.
- __Collaborative Solutions__: We encourage contributions and discussions, fostering a community-driven approach to solving these curious cases.

## Getting Started

To dive into DevGuru, start by exploring our [issues](http://gitlab:8081/dv_elop/dv_tools/dev_guru/-/issues). Each issue is tagged with relevant languages and concepts to help you find topics of interest quickly.

### Contributing

We welcome contributions from the community. Whether you have a curious case to share, a solution to propose, or insights to add to an existing discussion, your input is invaluable. Please, use a dedicaed issue template.

Open an issue labelled "gotcha", relevant existing labels and add @Mokshitha_R, as a reviewer.

### Navigating the Repository

- __Issues__: The heart of DevGuru. Explore, comment, and contribute to the discussions.
- __Code Snippets__: Found within some issues, these snippets highlight specific challenges and solutions.
- __Commit References__: For a deeper dive, some issues link directly to commits where you can see the context of changes made.
